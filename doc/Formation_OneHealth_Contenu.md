## Leçon 1 : Configuration Email Professionnel
**Durée:** 45 minutes
**Type:** Texte + Vidéo

### Objectifs d'apprentissage
- Configurer un email professionnel @onehealth.cm
- Comprendre les protocoles IMAP/SMTP
- Configurer la signature email professionnelle
- Gérer efficacement sa boîte mail

### Contenu détaillé

#### 1.1 Introduction aux emails professionnels
Les emails professionnels sont essentiels pour la communication officielle. L'adresse @onehealth.cm confère une crédibilité et une identité institutionnelle à vos communications.

**Avantages d'un email professionnel:**
- Crédibilité et professionnalisme
- Cohérence de la marque One Health
- Meilleure délivrabilité des emails
- Fonctionnalités avancées (calendrier partagé, contacts, etc.)

#### 1.2 Configuration sur Webmail (mail.infomaniak.com)
**Étapes de connexion:**
1. Accéder à https://mail.infomaniak.com
2. Entrer votre adresse email complète (prenom.nom@onehealth.cm)
3. Saisir le mot de passe fourni
4. Activer l'authentification à deux facteurs (recommandé)

**Interface Webmail:**
- Boîte de réception et dossiers
- Composition de messages
- Gestion des contacts
- Calendrier intégré
- Paramètres et préférences

#### 1.3 Configuration sur Outlook/Thunderbird
**Paramètres serveur Infomaniak:**
- **Serveur IMAP:** mail.infomaniak.com (Port 993, SSL/TLS)
- **Serveur SMTP:** mail.infomaniak.com (Port 587, STARTTLS)
- **Authentification:** Mot de passe normal

**Configuration Outlook:**
1. Fichier > Ajouter un compte
2. Configuration manuelle
3. Saisir les paramètres IMAP/SMTP
4. Tester la connexion

**Configuration Thunderbird:**
1. Menu > Paramètres des comptes > Ajouter un compte
2. Saisir nom, email et mot de passe
3. Configuration automatique ou manuelle
4. Valider

#### 1.4 Configuration sur Mobile (Android/iOS)
**Android (Gmail app):**
1. Paramètres > Ajouter un compte > Autre
2. Choisir IMAP
3. Configurer serveurs entrant/sortant
4. Synchroniser

**iOS (Mail):**
1. Réglages > Mail > Comptes > Ajouter un compte
2. Choisir "Autre" > Ajouter un compte Mail
3. Configurer IMAP
4. Activer Mail, Contacts, Calendriers

#### 1.5 Signature Email Professionnelle
**Modèle de signature recommandé:**
```
--

[Fonction/Titre]
Programme One Health Cameroun

Email: prenom.nom@onehealth.cm
Tél: +237 XXX XXX XXX
```

#### 1.6 Bonnes pratiques
- Répondre aux emails dans les 24-48h
- Utiliser des objets clairs et descriptifs
- Éviter les pièces jointes volumineuses (> 10 Mo)
- Archiver régulièrement les anciens emails

## Leçon 2 : Traitement d'images pour le site avec Photoshop
**Durée:** 1h30
**Type:** Vidéo + Pratique

### Objectifs d'apprentissage
- Redimensionner et optimiser les images pour le web
- Créer des miniatures et bannières
- Comprendre les formats d'image (JPEG, PNG, WebP)
- Utiliser les outils de base de Photoshop

### Contenu détaillé

#### 2.1 Formats d'images pour le web
**JPEG (.jpg):**
- Idéal pour: Photos, images complexes
- Compression: Avec perte (lossy)
- Taille: Légère à moyenne
- Utilisation: Thumbnails, photos d'actualités

**PNG (.png):**
- Idéal pour: Logos, images avec transparence
- Compression: Sans perte (lossless)
- Taille: Plus lourde
- Utilisation: Logos, icônes, graphiques

**WebP (.webp):**
- Idéal pour: Toutes les images web modernes
- Compression: Excellente (30% plus léger que JPEG)
- Support: Tous les navigateurs modernes
- Utilisation recommandée: Images du site OneHealth

#### 2.2 Dimensions recommandées pour OneHealth.cm
| Type d'image | Dimensions (px) | Format |
|--------------|-----------------|--------|
| Thumbnail article | 400 x 300 | WebP/JPEG |
| Image article full | 1200 x 800 | WebP/JPEG |
| Bannière hero | 1920 x 600 | WebP/JPEG |
| Logo partenaire | 200 x 100 | PNG |
| Photo expert | 300 x 300 | WebP/JPEG |
| Icône | 64 x 64 | PNG |

#### 2.3 Redimensionnement dans Photoshop
**Méthode 1: Image > Taille de l'image**
1. Ouvrir l'image (Ctrl+O)
2. Menu Image > Taille de l'image (Alt+Ctrl+I)
3. Décocher "Rééchantillonnage" si besoin de garder la qualité
4. Entrer les nouvelles dimensions
5. Choisir l'algorithme (Bicubique plus net pour réduction)

**Méthode 2: Outil Recadrage (C)**
1. Sélectionner l'outil Recadrage
2. Entrer dimensions dans la barre d'options
3. Définir la zone de recadrage
4. Valider (Entrée)

#### 2.4 Optimisation pour le web
**Exporter pour le web (Alt+Shift+Ctrl+S):**
1. Fichier > Exporter > Enregistrer pour le web
2. Choisir le format (JPEG, PNG, GIF)
3. Ajuster la qualité (60-80% pour JPEG)
4. Vérifier la taille du fichier (< 200 Ko idéalement)
5. Prévisualiser et enregistrer

**Conversion en WebP:**
1. Fichier > Exporter > Exporter sous
2. Choisir WebP comme format
3. Qualité: 75-85%
4. Enregistrer

#### 2.5 Création de thumbnails
**Processus standardisé:**
1. Ouvrir l'image source
2. Créer un nouveau document (400x300 px, 72 dpi, RGB)
3. Faire glisser l'image
4. Ajuster avec Transformation libre (Ctrl+T)
5. Appliquer un léger accentuation (Filtre > Renforcement > Accentuation)
6. Exporter en WebP

#### 2.6 Retouches basiques
**Luminosité/Contraste:**
- Image > Réglages > Luminosité/Contraste
- Ajuster pour une image équilibrée

**Niveaux (Ctrl+L):**
- Ajuster les tons clairs, moyens et foncés
- Utiliser l'histogramme comme guide

**Balance des couleurs:**
- Image > Réglages > Balance des couleurs
- Corriger les dominantes de couleur

#### 2.7 Actions automatisées (Batch Processing)
**Créer une action pour thumbnails:**
1. Fenêtre > Actions
2. Créer une nouvelle action "Thumbnail OneHealth"
3. Enregistrer les étapes (redimensionner, accentuer, exporter)
4. Arrêter l'enregistrement

**Traitement par lot:**
1. Fichier > Automatisation > Traitement par lots
2. Sélectionner l'action
3. Choisir le dossier source
4. Définir le dossier de destination
5. Lancer

### Exercice pratique
1. Télécharger 5 images de test
2. Créer des thumbnails 400x300
3. Créer une bannière 1920x600
4. Optimiser toutes les images en WebP
5. Vérifier que chaque fichier < 200 Ko

## Leçon 3 : Techniques d'écriture en ligne
**Durée:** 1h
**Type:** Texte + Exemples

### Objectifs d'apprentissage
- Rédiger pour le web de manière efficace
- Structurer un article pour le référencement (SEO)
- Adapter le ton et le style One Health
- Créer des titres accrocheurs

### Contenu détaillé

#### 3.1 Principes de la rédaction web
**La règle de la pyramide inversée:**
- Information la plus importante en premier
- Détails complémentaires ensuite
- Contexte et approfondissement à la fin

**Pourquoi?**
- Les lecteurs scannent (80% ne lisent pas tout)
- Les moteurs de recherche privilégient le début
- Mobile: écrans petits, attention limitée

#### 3.2 Structure d'un article optimisé
**Éléments essentiels:**
1. **Titre (H1):** 50-60 caractères, mot-clé principal
2. **Chapô/Introduction:** 2-3 phrases résumant l'essentiel
3. **Sous-titres (H2, H3):** Structure claire, mots-clés
4. **Paragraphes:** 3-4 phrases max, une idée par paragraphe
5. **Conclusion:** Récapitulatif + appel à l'action

**Exemple de structure:**
```
# Titre principal avec mot-clé
[Image à la une]

Introduction de 2-3 phrases...

## Premier sous-titre
Paragraphe explicatif...

### Sous-sous-titre si nécessaire
Détails...

## Deuxième sous-titre
Contenu...

## Conclusion
Résumé et appel à l'action
```

#### 3.3 Optimisation SEO de base
**Mots-clés:**
- Identifier 1-2 mots-clés principaux
- Les placer dans: titre, URL, premier paragraphe, sous-titres
- Densité: 1-2% du texte total
- Utiliser des synonymes et variantes

**Méta-description:**
- 150-160 caractères
- Résumer le contenu
- Inclure le mot-clé
- Inciter au clic

**URL optimisée:**
- Courte et descriptive
- Mots-clés séparés par des tirets
- Exemple: /actualites/formation-one-health-yaounde-2024

#### 3.4 Style rédactionnel One Health
**Ton recommandé:**
- Professionnel mais accessible
- Factuel et scientifiquement rigoureux
- Engageant sans être sensationnaliste
- Positif et orienté solutions

**Vocabulaire:**
- Éviter le jargon excessif
- Expliquer les termes techniques
- Utiliser des exemples concrets
- S'adresser au lecteur ("vous")

#### 3.5 Titres et accroches
**Techniques de titres efficaces:**
1. **Chiffres:** "5 mesures pour prévenir la rage"
2. **Questions:** "Comment signaler une zoonose?"
3. **Comment/Guide:** "Guide complet de la surveillance épidémiologique"
4. **Actualité:** "Formation One Health: 50 cadres formés à Douala"

**À éviter:**
- Titres trop longs (> 70 caractères)
- Titres vagues ou génériques
- Majuscules excessives
- Ponctuation multiple (!!!, ???)

#### 3.6 Intégration multimédia
**Images:**
- Une image principale (featured image)
- Images dans le corps si pertinent
- Toujours ajouter un texte alternatif (alt)
- Créditer les sources

**Vidéos:**
- Intégrer via YouTube/Vimeo
- Ajouter une description
- Prévoir une transcription si possible

**Documents:**
- Proposer en téléchargement (PDF)
- Résumer le contenu du document
- Indiquer la taille du fichier

#### 3.7 Checklist avant publication
- [ ] Titre optimisé (50-60 caractères)
- [ ] Méta-description rédigée (150-160 caractères)
- [ ] URL propre et avec mot-clé
- [ ] Image à la une de qualité
- [ ] Alt text sur toutes les images
- [ ] Liens internes vers d'autres articles
- [ ] Catégorie assignée
- [ ] Relecture orthographique effectuée
- [ ] Aperçu vérifié sur mobile

### Exercice pratique
Rédiger un article de 400 mots sur le thème:
"Lancement d'une nouvelle formation One Health au Cameroun"

## Leçon 4 : Architecture Plateforme / Site Web
**Durée:** 1h
**Type:** Texte + Schémas

### Objectifs d'apprentissage
- Comprendre l'architecture globale de OneHealth.cm
- Identifier les différents modules et leur rôle
- Naviguer efficacement dans la plateforme
- Comprendre les interactions entre modules

### Contenu détaillé

#### 4.1 Vue d'ensemble de la plateforme
**OneHealth.cm est composé de 4 plateformes principales:**

```
┌─────────────────────────────────────────────────────────────┐
│                    ONEHEALTH.CM                             │
├─────────────┬─────────────┬─────────────┬─────────────────┤
│   SITE WEB  │    OHWR     │   COHRM     │   E-LEARNING    │
│   VITRINE   │   MAPPING   │   RUMEURS   │   FORMATION     │
├─────────────┴─────────────┴─────────────┴─────────────────┤
│              PANNEAU D'ADMINISTRATION                      │
│                 admin.onehealth.cm                         │
└─────────────────────────────────────────────────────────────┘
```

#### 4.2 Site Web Vitrine (onehealth.cm)
**Objectif:** Présenter One Health, informer le public, publier l'actualité

**Pages principales:**
- **Accueil:** Page dynamique avec sections configurables
- **À propos:** Présentation du programme
- **Actualités:** Articles et communiqués
- **Zoonoses:** Information sur les maladies zoonotiques
- **Piliers:** Les 3 piliers One Health
- **Partenaires:** Logos et liens partenaires
- **Contact:** Formulaire et coordonnées

**Fonctionnalités:**
- Multilingue (Français/Anglais)
- Responsive (mobile, tablette, desktop)
- SEO optimisé
- Newsletter intégrée

#### 4.3 OHWR Mapping (onehealth.cm/ohwr)
**Objectif:** Cartographier les ressources One Health au Cameroun

**Composants:**
- **Experts:** Base de données des experts One Health
- **Organisations:** Institutions et partenaires
- **Ressources matérielles:** Équipements et infrastructures
- **Documents:** Guides, protocoles, publications
- **Carte interactive:** Visualisation géographique

**Cas d'utilisation:**
- Trouver un expert dans une région
- Identifier les organisations actives
- Accéder aux ressources documentaires
- Planifier des interventions

#### 4.4 COHRM - Gestion des Rumeurs
**Objectif:** Surveiller et gérer les rumeurs sanitaires

**Flux de travail:**
```
Signalement → Réception → Investigation → Confirmation/Infirmation → Clôture
     ↑            ↓
   Sources    Classification
   - SMS      - Priorité
   - Web      - Catégorie
   - App      - Région
```

**Sources de données:**
- SMS codés (format standardisé)
- Scan web et réseaux sociaux
- Application mobile
- Signalement direct

#### 4.5 OH E-Learning (onehealth.cm/oh-elearning)
**Objectif:** Former les acteurs One Health en ligne

**Structure:**
```
Parcours (Diplôme)
    └── Cours
          └── Modules
                └── Leçons
                      └── Quiz
                            └── Certificat
```

**Fonctionnalités apprenants:**
- Inscription aux cours
- Suivi de progression
- Quiz d'évaluation
- Certificats téléchargeables

**Fonctionnalités admin:**
- Création de cours
- Gestion des inscriptions
- Suivi des statistiques
- Personnalisation des certificats

#### 4.6 Panneau d'Administration (admin.onehealth.cm)
**Accès:** Réservé aux administrateurs et rédacteurs

**Modules disponibles:**
1. **Dashboard:** Vue d'ensemble et statistiques
2. **Articles:** Gestion du contenu éditorial
3. **Pages:** Pages statiques du site
4. **Médias:** Bibliothèque d'images et fichiers
5. **Menus:** Navigation du site
6. **Catégories:** Organisation du contenu
7. **Utilisateurs:** Gestion des comptes
8. **E-Learning:** Administration des formations
9. **OHWR:** Gestion du mapping
10. **COHRM:** Gestion des rumeurs
11. **Newsletter:** Campagnes email
12. **Paramètres:** Configuration globale

## Leçon 6 : Gestion de Contenu dans le site web
**Durée:** 2h
**Type:** Vidéo + Pratique

### Objectifs d'apprentissage
- Créer et publier des articles
- Gérer les médias (images, documents)
- Organiser le contenu en catégories
- Utiliser le Page Builder

### Contenu détaillé

#### 6.1 Gestion des Articles
**Créer un nouvel article:**
1. Menu > Articles > Nouveau
2. Remplir les champs:
   - **Titre:** Titre de l'article
   - **Contenu:** Éditeur riche (texte, images, liens)
   - **Extrait:** Résumé court pour les listes
   - **Image à la une:** Thumbnail principale
   - **Catégorie:** Classer l'article
   - **Tags:** Mots-clés supplémentaires

**Éditeur de contenu:**
- Barre d'outils: Gras, Italique, Listes, Liens
- Insertion d'images depuis la médiathèque
- Insertion de vidéos (YouTube, Vimeo)
- Tableaux et mise en forme avancée

**Statuts d'article:**
- **Brouillon:** En cours de rédaction
- **En attente:** Soumis pour validation
- **Publié:** Visible sur le site
- **Planifié:** Publication automatique future
- **Archivé:** Retiré du site mais conservé

**SEO de l'article:**
- Méta-titre (différent du titre si besoin)
- Méta-description
- URL personnalisée (slug)
- Mots-clés

#### 6.2 Gestion des Médias
**Accéder à la médiathèque:**
- Menu > Médias
- Ou depuis l'éditeur d'article

**Uploader des fichiers:**
1. Cliquer "Ajouter"
2. Glisser-déposer ou sélectionner
3. Formats acceptés: JPEG, PNG, WebP, PDF, DOCX, XLSX

**Organiser les médias:**
- Recherche par nom
- Filtrer par type (Images, Documents, Vidéos)
- Filtrer par date

**Modifier un média:**
- Cliquer sur le média
- Modifier: Titre, Alt text, Description
- Copier l'URL pour utilisation externe

**Bonnes pratiques:**
- Nommer clairement les fichiers avant upload
- Toujours remplir le texte alternatif (accessibilité + SEO)
- Supprimer les médias non utilisés

#### 6.3 Catégories et Tags
**Gérer les catégories:**
- Menu > Catégories
- Créer: Nom, Description, Image, Couleur
- Hiérarchie: Catégories parentes/enfants

**Utiliser les tags:**
- Mots-clés libres
- Permettent une navigation transversale
- Pas de hiérarchie

**Différence catégories/tags:**
| Catégories | Tags |
|------------|------|
| Structure principale | Mots-clés complémentaires |
| Hiérarchiques | Plats |
| Obligatoire (1 min) | Optionnels |
| Peu nombreuses | Peuvent être nombreux |

#### 6.4 Gestion des Pages
**Pages vs Articles:**
- **Pages:** Contenu statique (À propos, Contact)
- **Articles:** Contenu daté (Actualités)

**Créer une page:**
1. Menu > Pages > Nouvelle
2. Titre et contenu
3. Modèle de page (si disponible)
4. Publier

#### 6.5 Page Builder (Accueil)
**Accéder au Page Builder:**
- Menu > Page Builder
- Ou via le bouton dédié

**Types de sections disponibles:**
- **Hero:** Bannière principale avec titre et CTA
- **Texte:** Paragraphes et mise en forme
- **Image/Galerie:** Images uniques ou carrousels
- **Vidéo:** Intégration YouTube/Vimeo
- **CTA:** Appels à l'action
- **Cartes:** Grilles d'éléments
- **Statistiques:** Chiffres clés
- **Partenaires:** Logos partenaires
- **Actualités:** Derniers articles
- **Piliers:** Présentation One Health
- **Équipe:** Membres de l'équipe
- **FAQ:** Questions fréquentes

**Gérer les sections:**
1. Cliquer "Ajouter une section"
2. Choisir le type
3. Configurer le contenu (FR/EN)
4. Réordonner par glisser-déposer
5. Activer/désactiver individuellement
6. Prévisualiser
7. Publier

#### 6.6 Gestion des Menus
**Créer/Modifier un menu:**
1. Menu > Menus
2. Sélectionner le menu (Header, Footer)
3. Ajouter des éléments:
   - Pages existantes
   - Catégories
   - Liens personnalisés
4. Réordonner par glisser-déposer
5. Définir la hiérarchie (sous-menus)
6. Enregistrer

# MODULE 4 : COHRM - GESTION DES RUMEURS (Leçons 11-14)

## Leçon 11 : COHRM - Introduction
**Durée:** 45 min
**Type:** Texte + Démonstration

### Objectifs d'apprentissage
- Comprendre le système COHRM
- Connaître les sources de signalement
- Naviguer dans l'interface COHRM

### Contenu détaillé

#### 11.1 Qu'est-ce que COHRM?
**COHRM = Community One Health Rumor Management**

**Objectif:**
Détecter, vérifier et gérer les rumeurs sanitaires pour une réponse rapide aux menaces potentielles.

**Pourquoi c'est important:**
- Détection précoce des épidémies
- Lutte contre la désinformation
- Coordination des réponses
- Traçabilité des signalements

#### 11.2 Flux de travail COHRM
```
SIGNALEMENT → RÉCEPTION → TRIAGE → INVESTIGATION → DÉCISION → CLÔTURE
                                                    ↓
                                          Confirmé / Fausse alerte
```

#### 11.3 Sources de signalement
**1. SMS codés:**
Format: CODE REGION DETAILS
Exemple: EP CE Cas suspects de fièvre hémorragique à Bertoua

**Codes SMS:**
| Code | Signification |
|------|---------------|
| EP | Épidémie suspectée |
| ZO | Cas de zoonose |
| MA | Maladie animale |
| EC | Eau contaminée |
| AL | Aliment suspect |
| EN | Problème environnemental |
| DC | Décès suspect |
| AU | Autre signalement |

**2. Scan Web/Réseaux sociaux:**
- Surveillance automatique
- Mots-clés prédéfinis
- Sources: Twitter, Facebook, forums, médias

**3. Application mobile:**
- Signalement avec géolocalisation
- Photos jointes
- Formulaire guidé

**4. Signalement direct:**
- Hotline téléphonique
- Email
- Visite en personne

#### 11.4 Interface COHRM
**Dashboard:**
- Rumeurs en attente
- Rumeurs en cours d'investigation
- Alertes actives
- Statistiques

**Liste des rumeurs:**
- Tableau avec filtres
- Colonnes: ID, Titre, Priorité, Statut, Région, Date
- Actions rapides

## Leçon 12 : Signalement Rumeurs
**Durée:** 1h
**Type:** Vidéo + Pratique

### Objectifs d'apprentissage
- Créer un signalement de rumeur
- Utiliser le décodeur SMS
- Géolocaliser une rumeur
- Classifier par priorité et catégorie

### Contenu détaillé

#### 12.1 Créer une nouvelle rumeur
**Accès:** COHRM > Rumeurs > Nouvelle rumeur

**Informations essentielles:**
1. **Titre:** Description courte et claire
2. **Description:** Détails complets du signalement
3. **Catégorie:** Type de menace
4. **Priorité:** Niveau d'urgence
5. **Source:** Origine du signalement

#### 12.2 Catégories de rumeurs
| Catégorie | Description | Exemples |
|-----------|-------------|----------|
| Épidémie | Propagation de maladie | Choléra, COVID |
| Zoonose | Transmission animal-homme | Rage, Grippe aviaire |
| Sécurité alimentaire | Contamination aliments | Intoxication |
| Eau contaminée | Problème eau potable | Diarrhées |
| Maladie animale | Affection du bétail | Peste porcine |
| Environnement | Pollution, déchets | Déversement |
| Autre | Non catégorisé | - |

#### 12.3 Niveaux de priorité
| Priorité | Critères | Délai de traitement |
|----------|----------|---------------------|
| Critique | Décès, propagation rapide | Immédiat (< 2h) |
| Haute | Plusieurs cas, zone sensible | Urgent (< 12h) |
| Moyenne | Cas isolé, situation stable | 24-48h |
| Basse | Signalement vague, faible risque | 72h |

#### 12.4 Géolocalisation
**Localiser la rumeur:**
1. Sélectionner la Région
2. Sélectionner le District
3. Entrer la Localité
4. Option: Coordonnées GPS précises
5. Option: Cliquer sur la carte

**Pourquoi géolocaliser:**
- Cartographie des menaces
- Identification des clusters
- Coordination des équipes terrain

#### 12.5 Informations complémentaires
**Informateur:**
- Nom (optionnel)
- Téléphone (pour recontact)
- Email
- Relation avec le cas (témoin, victime, professionnel)

**Détails sanitaires:**
- Nombre de personnes affectées (estimé)
- Symptômes observés
- Animaux impliqués
- Actions déjà prises

#### 12.6 Utiliser le décodeur SMS
**Accès:** COHRM > Décodeur SMS

**Processus:**
1. Coller le message SMS reçu
2. Cliquer "Décoder"
3. Le système extrait:
   - Code → Catégorie
   - Région
   - Détails
4. Vérifier et compléter
5. Créer la rumeur

**Exemple:**
```
Message: EP CE Plusieurs cas de fièvre avec saignements signalés à Bertoua
Décodage:
- Catégorie: Épidémie
- Priorité suggérée: Critique
- Région: Centre-Est
- Détails: Plusieurs cas de fièvre avec saignements signalés à Bertoua
```

## Leçon 13 : Traitement Rumeurs
**Durée:** 1h
**Type:** Vidéo + Pratique

### Objectifs d'apprentissage
- Traiter une rumeur de bout en bout
- Documenter les investigations
- Changer les statuts appropriés
- Clôturer avec les conclusions

### Contenu détaillé

#### 13.1 Statuts des rumeurs
| Statut | Description | Actions |
|--------|-------------|---------|
| En attente | Nouveau signalement | Trier, assigner |
| En investigation | Vérification en cours | Documenter, enquêter |
| Confirmé | Menace avérée | Alerter, coordonner |
| Fausse alerte | Signalement infondé | Documenter, clôturer |
| Clôturé | Traitement terminé | Archiver |

#### 13.2 Processus de traitement
**Étape 1: Triage (En attente → En investigation)**
1. Ouvrir la rumeur
2. Évaluer la crédibilité
3. Vérifier les informations de base
4. Assigner un niveau de priorité
5. Passer en "En investigation"

**Étape 2: Investigation**
1. Contacter l'informateur si possible
2. Vérifier auprès des sources officielles
3. Envoyer une équipe terrain si nécessaire
4. Documenter chaque action

**Étape 3: Décision**
- Si confirmé: Déclencher les protocoles d'alerte
- Si fausse alerte: Documenter les raisons
- Si incertain: Continuer l'investigation

**Étape 4: Clôture**
1. Rédiger le résumé final
2. Lister les actions entreprises
3. Changer le statut final
4. Archiver

#### 13.3 Documenter les actions
**Dans le champ "Actions prises":**
- Date et heure de chaque action
- Personne responsable
- Action effectuée
- Résultat obtenu

**Exemple de documentation:**
```
[15/01/2024 10:30] - Dr. Kamga
Contact téléphonique avec l'informateur.
Confirme 5 cas de diarrhée sévère dans le village X.

[15/01/2024 14:00] - Équipe terrain
Visite du village. Prélèvements d'eau effectués.
12 cas identifiés, 2 hospitalisés.

[16/01/2024 09:00] - Laboratoire
Résultats: Présence de Vibrio cholerae.
Alerte choléra confirmée.
```

#### 13.4 Notifications automatiques
**Notifications par priorité:**
| Priorité | Email | SMS | Push |
|----------|-------|-----|------|
| Critique | ✓ | ✓ | ✓ |
| Haute | ✓ | ✓ | - |
| Moyenne | ✓ | - | - |
| Basse | - | - | - |

**Configurer les notifications:**
COHRM > Paramètres > Notifications

## Leçon 20 : Pratique - Scénario complet multi-modules
**Durée:** 2h30
**Type:** Atelier pratique intégré

### Objectifs
- Intégrer toutes les compétences acquises
- Simuler un cas réel de bout en bout
- Travailler en équipe

### Scénario

**Contexte:**
Une épidémie de grippe aviaire est suspectée dans la région du Nord. Vous devez:
1. Gérer l'alerte dans COHRM
2. Identifier les experts et ressources via OHWR
3. Publier une communication sur le site
4. Envoyer une newsletter d'alerte
5. Préparer une formation e-learning de sensibilisation

### Étapes détaillées

#### Partie 1: COHRM (30 min)
1. Créer la rumeur "Grippe aviaire suspectée - Nord"
   - Catégorie: Zoonose
   - Priorité: Haute
   - Géolocaliser à Garoua
2. Documenter l'investigation
3. Confirmer l'alerte
4. Lister les actions recommandées

#### Partie 2: OHWR (20 min)
1. Rechercher les experts en santé animale dans le Nord
2. Identifier les organisations partenaires
3. Localiser les laboratoires disponibles
4. Télécharger les protocoles grippe aviaire

#### Partie 3: Site Web (40 min)
1. Rédiger un communiqué de presse
   - Titre: "Alerte grippe aviaire dans le Nord: mesures de précaution"
   - 400 mots, SEO optimisé
2. Créer/sélectionner une image appropriée
3. Publier l'article
4. Vérifier sur le site public

#### Partie 4: Newsletter (20 min)
1. Créer une campagne "Alerte Grippe Aviaire"
2. Rédiger un email court et informatif
3. Sélectionner la liste "Alertes sanitaires"
4. Programmer l'envoi immédiat
5. Vérifier les statistiques

#### Partie 5: E-Learning (40 min)
1. Créer un mini-cours "Prévention Grippe Aviaire"
2. Ajouter 2 modules:
   - Module 1: Comprendre la grippe aviaire
   - Module 2: Mesures de protection
3. Créer 1 leçon texte par module
4. Créer un quiz de 5 questions
5. Publier le cours

### Évaluation finale

|---------|--------|
| Rumeur correctement gérée | /20 |
| Ressources OHWR identifiées | /15 |
| Article bien rédigé | /25 |
| Newsletter envoyée | /15 |

| **Total** | **/100** |
